The bot runs in the cloud on Modal (free tier). Every few minutes, it:

  1. Reads the latest signal from Prof Adam's Portfolio Signals channel via TRW's API

05:00–00:00 → sends you a Slack notification to approve before trading

Step 1: Get Your TRW Session Token

The bot needs to read messages from the signal channel. TRW uses a session token for authentication.

  1. Open The Real World in Chrome or Brave
  2. Navigate to Adam's Portfolio Signals channel
  3. Press F12 to open DevTools
  4. Click the Network tab
  5. Click the Fetch/XHR filter button
  6. Reload the page (F5)
  7. Click on any request in the list
  8. In the Headers tab, scroll down to Request Headers
  9. Find x-session-token — copy the entire value
NEVER share your session token with anyone. Copy it to your clipboard and use it ONLY in the manager GUI (python manage.py). Do not paste it into Discord, AI chats, websites, or anywhere else. The only places your token should ever exist are:
  • The manager GUI running on your local machine
  • Your local .env file (if you know what you're doing)
If anyone asks you for your session token — for any reason — they are trying to scam you.

How long does it last? Session tokens appear to be long-lived (days to weeks). If the bot stops working, you may need to re-grab the token. The bot will notify you on Slack if this happens.

Step 2: Set Up Hyperliquid API

The bot needs an API wallet to place trades on your behalf. API wallets cannot withdraw funds — they can only place and cancel orders.

  1. Go to app.hyperliquid.xyz and make sure your wallet is connected and you have funds deposited
  2. Click More → API in the top menu
  3. Give your API wallet a name (e.g., "signal-bot") — this is important because generating a new unnamed API wallet will deregister any previous unnamed one
  4. Click Generate, then click Authorize API Wallet
  5. Set the Days Valid field — you can go up to 180 days max (recommended: set it to the maximum)
  6. Copy and securely store the Private Key it gives you — this is what the bot uses to sign trades
  7. Click Authorize and sign the transaction in your wallet (MetaMask will pop up — don't skip this step or the key won't work)
  8. Grab your Master Account Address from the dropdown next to your connected wallet in the top right corner of the Hyperliquid app
You end up with three pieces of info:
  • API Wallet Address — the public address of the generated wallet (you don't need this for the bot)
  • Private Key — the secret key for signing trades (goes into your .env as HYPERLIQUID_API_PRIVATE_KEY)
  • Master Account Address — your main wallet's public address (goes into your .env as HYPERLIQUID_MASTER_ACCOUNT_ADDRESS)
NEVER share these keys with anyone. Copy them to your clipboard and use them ONLY in the manager GUI (python manage.py). Do not paste them into Discord, AI chats, websites, or anywhere else. The only places your keys should ever exist are:
  • The manager GUI running on your local machine
  • Your local .env file (if you know what you're doing)
If anyone asks you for your private key or session token — for any reason — they are trying to scam you.
Key things to remember:
  • The API wallet can place and cancel orders but cannot withdraw funds
  • The bot uses your master account address to query your positions and balance, and the API private key to sign trades
  • If you generate a new unnamed API wallet, it deregisters the previous unnamed one — always name your wallets if you want multiple
  • The key expires after the validity period you set — set it to 180 days and put a reminder in your calendar to regenerate it

Step 3: Set Up Modal (Free Cloud Hosting)

Modal runs your bot 24/7 in the cloud for free. You don't need to leave your computer on.

Create a Modal account

  1. Go to modal.com
  2. Click "Sign Up" — you can use Google or GitHub to sign up
  3. No credit card required — the free tier is more than enough for this bot

Install the Modal CLI

Open a terminal/command prompt and run:

pip install modal

Connect Modal to your account

Run this command and it will open a browser window to authenticate:

Click "Approve" in the browser when prompted. You should see a success message in your terminal like:

Token verified successfully!
Token written to ~/.modal.toml

Verify it works

This should print your Modal workspace name (e.g., your-username-1234). If you see an error, try running modal token new again.

That's it! Modal is now set up. The manager (python manage.py) will handle creating secrets and deploying the bot to your Modal account automatically.

FAQ

Why perps instead of spot?

ETH, PAXG, and XAUT don't have spot markets on Hyperliquid — they only exist as perps. Using 1x leverage perps gives identical economic exposure to holding spot. All positions use 1x cross leverage (no amplified risk).

What if the signal says "No change"?

The bot detects this and does nothing. It sends a Slack notification confirming "No change" so you know it's working.

What if the professor posts other messages (not signals)?

The bot filters for messages containing "Portfolio Signal Update" from Prof Adam specifically. It ignores everything else.

What if my internet goes down?

The bot runs in Modal's cloud, not on your computer. Your internet doesn't matter once deployed.

What if Hyperliquid is down?

The bot will fail to execute trades and send you a Slack error notification. It will retry on the next poll cycle.

Can I stop the bot temporarily?

Yes: modal app stop signal-bot
Restart: modal deploy modal_signal_bot.py

Can I change the schedule?

Yes — edit the should_poll_now() function in modal_signal_bot.py and redeploy.

Can I use this for other signal channels?

Yes — change TRW_SIGNAL_CHANNEL_ID and TRW_PROF_ADAM_USER_ID in your .env to match a different channel/professor. You may need to adjust the signal parser if the format is different.
